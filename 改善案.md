# Role
あなたは熟練したフロントエンドエンジニアであり、TampermonkeyなどのUserScript開発のエキスパートです。

# Task
提供する「Pixiv Discovery Anti-Reload」スクリプトをリファクタリングし、コード品質、パフォーマンス、保守性を向上させてください。
現在の機能（キャッシュによる作品リスト維持、スクロール復元）は維持したまま、以下の改善点を実装してください。

# Requirements
1. **スクロール復元の最適化**:
   - 現在の `setTimeout` を再帰的に呼ぶ実装は描画サイクルと同期していないため、`requestAnimationFrame` を使用したロジックに変更し、よりスムーズな復元動作にしてください。

2. **レガシーコードの削除**:
   - `getApiCache` 関数内にある「旧形式（v1）からのデータ移行ロジック」は不要になったため削除し、コードをシンプルにしてください。

3. **設定値の集約**:
   - `RESTORE_WINDOW`、`CACHE_DURATION`、`API_CACHE_MAX_ENTRIES` などの定数を、スクリプト冒頭の `CONFIG` オブジェクトにまとめて定義し、調整しやすくしてください。

4. **モダンな記述と可読性**:
   - 全体的に最新のJavaScript記法（ES6+）を適用してください。
   - 適切なJSDocコメントを追加し、各関数の役割を明確にしてください。
   - 不要なネストを解消し（Early Returnの活用など）、可読性を高めてください。

5. **堅牢性の維持**:
   - `sessionStorage` へのアクセスやJSONパース時の `try-catch` エラーハンドリングは維持（または強化）してください。
